Objective:
Update the existing BROWZ Academy landing page to support a marketplace model by introducing a dedicated provider entry point.

Header / Navigation Changes

Add a new navigation item in the header labeled “Become a Provider”

Position it alongside existing navigation items

Style it consistently with the current design system (fonts, colors, spacing)

On click, route the user to a new page: /become-a-provider

New Page: Provider Landing Page

Create a new standalone landing page specifically for course providers (individual instructors and institutions).

Page Purpose

Clearly explain how providers can teach, publish, and earn on the platform

Support both individual instructors and institutions

Encourage qualified providers to apply

Page Structure & Sections

Hero Section

Headline focused on providers (e.g. “Teach. Publish. Earn.”)

Short subtext explaining that providers can reach learners and monetize expertise

Primary CTA: “Join as a Provider”

Secondary CTA (optional): “Learn how it works”

Who Can Join

Two clear cards or sections:

Individual Instructors (experts, professionals)

Institutions & Academies (schools, training bodies, faculties)

Brief descriptions for each

How It Works

Simple 3–4 step flow:

Apply as a provider

Get verified

Publish courses

Earn from enrollments

Trust & Quality

Mention that providers are verified to ensure quality for learners

Keep language high-level (no internal TXM or admin terminology)

Call to Action

Prominent “Apply to Become a Provider” button

CTA routes to existing or placeholder provider onboarding flow

Technical / UX Notes

Do not alter the existing learner landing page content

Ensure routing works correctly between learner landing page and provider landing page

Reuse existing components where possible for consistency

Ensure the provider page is responsive

Maintain current branding colors and typography

Success Criteria

Learners and providers have clearly separated entry paths

The platform communicates its marketplace nature without confusion

Providers immediately understand whether the platform is for them
